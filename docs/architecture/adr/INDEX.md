## ADR Lifecycle

```
💡 Proposed → ✅ Approved → 🔄 Superseded (by new ADR)
                     ↓
                ⛔ Deprecated
```

## Decision Making Process

1. **Proposal** — Any principal can propose an ADR
2. **Review** — ADR is reviewed by the engineering team
3. **Vote** — Simple majority required for approval
4. **Approved** — Becomes binding architectural constraint
5. **Superseded** — When a new ADR replaces a previous decision
